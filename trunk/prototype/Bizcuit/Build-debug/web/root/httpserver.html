The code is certainly useful for understanding sockets and threading. But manual thread scheduling using "new Thread" and "Thread.Start" is not the recommended approach for building a scalable web server. If the web server receives 200 requests simultanenous, 200 threads will be spawned on the server. With so many threads, more time resources will be consumed in context switching rather than useful processing. Instead, one should look into using asynchronous sockets that make efficient use of the ThreadPool.

Just a comment for those considering using this a start for building a "real" server. The code is useful nonetheless.
